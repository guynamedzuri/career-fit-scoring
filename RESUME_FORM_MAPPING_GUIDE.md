# 이력서 양식 매핑 가이드

이 문서는 `resume_form.docx`에서 추출한 모든 칸의 위치 정보를 제공합니다.
각 칸에 어떤 데이터가 들어가야 하는지 알려주시면 `resumeMapping.ts`의 매핑 설정을 업데이트하겠습니다.

## 테이블 구조 요약

총 **7개의 테이블**이 있습니다:

- **테이블 0**: 일반정보/민감정보 구분 (2행)
- **테이블 1**: 기본 인적사항 (9행)
- **테이블 2**: 학력사항 (8행)
- **테이블 3**: 경력사항 (7행)
- **테이블 4**: 어학/자격사항/수상경력 (9행)
- **테이블 5**: 자기소개/지원동기/성과목표/장래포부 (8행)
- **테이블 6**: 경력기술 상세 (17행)

## 상세 구조

### 테이블 1: 기본 인적사항 (tableIndex: 1)

#### 헤더 행 (행 0)
- 열 0-3: "▣ 기본 인적사항"
- 열 4: "지원분야"
- 열 5: 빈칸

#### 데이터 행들

**행 1: 성명 (한글/한문)**
- 열 0: "성명"
- 열 1-3: "(한글) (한문)" (입력 필드)
- 열 4: "희망연봉"
- 열 5: "(만원)" (입력 필드)

**행 2: 성명 (영문)**
- 열 0: "성명"
- 열 1-4: "(영문)" (입력 필드)
- 열 5: 빈칸

**행 3: 생년월일 / e-mail**
- 열 0: "생년월일"
- 열 1-2: 빈칸 (입력 필드)
- 열 3: "e-mail"
- 열 4-6: 빈칸 (입력 필드)

**행 4: 현주소**
- 열 0: "현주소"
- 열 1-5: 빈칸 (입력 필드)

**행 5: 자택전화 / 보훈대상**
- 열 0: "자택전화"
- 열 1-2: 빈칸 (입력 필드)
- 열 3: "보훈대상"
- 열 4: "대상/비대상"
- 열 5: 빈칸 (입력 필드)

**행 6: 이동전화 / 장애인정**
- 열 0: "이동전화"
- 열 1-2: 빈칸 (입력 필드)
- 열 3: "장애인정"
- 열 4: "대상/비대상"
- 열 5: 빈칸 (입력 필드)

**행 7: 병역사항 (헤더)**
- 열 0: "병역사항"
- 열 1: "군필여부/종류"
- 열 2: "면제사유"
- 열 3: "계급"
- 열 4: "입대년월"
- 열 5: "제대년월"

**행 8: 병역사항 (데이터)**
- 열 0: "병역사항"
- 열 1-5: 빈칸 (입력 필드)

### 테이블 2: 학력사항 (tableIndex: 2)

#### 헤더 행 (행 1)
- 열 0: "입학년월"
- 열 1: "졸업년월"
- 열 2: "학교명"
- 열 3: "전공"
- 열 4: "학점"
- 열 5: "소재지"
- 열 6: "졸업구분"

#### 데이터 행들 (행 2-7)
- 행 2-4: "대학교" (예시 텍스트)
- 행 5-6: "대학원" (예시 텍스트)
- 행 7: 빈 행

각 행의 열들은 빈칸이거나 예시 텍스트입니다.

### 테이블 3: 경력사항 (tableIndex: 3)

#### 헤더 행 (행 1)
- 열 0: "입사년월"
- 열 1: "퇴사년월"
- 열 2: "회사명"
- 열 3: "근무부서"
- 열 4: "직위"
- 열 5: "연봉"
- 열 6: "이직사유"

#### 데이터 행들 (행 2-6)
모든 행이 빈칸입니다 (입력 필드).

### 테이블 4: 어학/자격사항/수상경력 (tableIndex: 4)

#### 헤더 행 (행 1)
- 열 0: "어학종류(명)"
- 열 1: "점수/등급"
- 열 2: "취득일자"
- 열 3: "자격명"
- 열 4: "등급"
- 열 5: "발행기관"

#### 데이터 행들 (행 2-4)
모든 행이 빈칸입니다 (입력 필드).

#### 추가 헤더 (행 5)
- 열 0: "해외연수국가"
- 열 1: "거주기간"
- 열 2: "거주목적"
- 열 3: "수상명"
- 열 4: "수상기관"
- 열 5: "수상내역"

#### 데이터 행들 (행 6-8)
모든 행이 빈칸입니다 (입력 필드).

### 테이블 5: 자기소개/지원동기/성과목표/장래포부 (tableIndex: 5)

#### 항목별 구조
- **행 0-1**: 자기소개 (700자이내)
  - 열 0: "자기소개 (700자이내)"
  - 열 1: 설명 텍스트 또는 빈칸 (입력 필드)

- **행 2-3**: 지원동기 (700자이내)
  - 열 0: "지원동기 (700자이내)"
  - 열 1: 설명 텍스트 또는 빈칸 (입력 필드)

- **행 4-5**: 성과목표 달성경험 (700자이내)
  - 열 0: "성과목표 달성경험 (700자이내)"
  - 열 1: 설명 텍스트 또는 빈칸 (입력 필드)

- **행 6-7**: 장래포부 (700자이내)
  - 열 0: "장래포부 (700자이내)"
  - 열 1: 설명 텍스트 또는 빈칸 (입력 필드)

### 테이블 6: 경력기술 상세 (tableIndex: 6)

이 테이블은 반복되는 구조를 가지고 있습니다:

#### 첫 번째 경력 블록 (행 1-4)
- **행 1**: 헤더
  - 열 0: "입사년월"
  - 열 1: "퇴사년월"
  - 열 2: "회사명"
  - 열 3: "근무부서"
  - 열 4: "직위"
  - 열 5: "연봉"
  - 열 6: "이직사유"

- **행 2**: 빈 행 (입력 필드)

- **행 3**: "상세내용(담당업무/프로젝트 등)" (7개 열 병합)

- **행 4**: "* 임직원수 : * 매출액 :" (7개 열 병합)

#### 두 번째 경력 블록 (행 5-8)
- **행 5**: 헤더 (행 1과 동일)
- **행 6**: 빈 행 (입력 필드)
- **행 7**: "상세내용(담당업무/프로젝트 등)"
- **행 8**: "* 임직원수 : * 매출액 :"

#### 세 번째 경력 블록 (행 9-12)
- **행 9**: 헤더 (행 1과 동일)
- **행 10**: 빈 행 (입력 필드)
- **행 11**: "상세내용(담당업무/프로젝트 등)"
- **행 12**: "* 임직원수 : * 매출액 :"

## 매핑 정보 요청

다음 형식으로 각 칸에 어떤 데이터가 들어가야 하는지 알려주세요:

```
테이블 [번호], 행 [번호], 열 [번호]: [데이터 필드명]
예: 테이블 1, 행 1, 열 1: 이름 (한글)
```

또는 더 간단하게:

```
- 이름 (한글): 테이블 1, 행 1, 열 1
- 이름 (영문): 테이블 1, 행 2, 열 1
- 생년월일: 테이블 1, 행 3, 열 1
- 이메일: 테이블 1, 행 3, 열 4
- 주소: 테이블 1, 행 4, 열 1
- 전화번호: 테이블 1, 행 6, 열 1
...
```
