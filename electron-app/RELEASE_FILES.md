# GitHub Release에 업로드할 파일

## 필수 파일 (2개만!)

### 1. 설치 파일 (.exe)
```
이력서 적합도 평가 시스템 Setup 1.0.1.exe
```
- 위치: `dist-installer/` 폴더
- 용도: 사용자가 다운로드해서 설치할 파일
- 크기: 보통 100MB 이상

### 2. 업데이트 메타데이터 (latest.yml)
```
latest.yml
```
- 위치: `dist-installer/` 폴더
- 용도: 자동 업데이트 기능이 새 버전을 확인하는데 필요
- 크기: 매우 작음 (몇 KB)
- 중요: 이 파일이 없으면 자동 업데이트가 작동하지 않습니다!

## 올리지 말아야 할 것

### ❌ win-unpacked 폴더
- 올릴 필요 없습니다!
- 설치 파일(.exe)이 이미 이 내용을 포함하고 있습니다
- 사용자는 설치 파일만 다운로드하면 됩니다

### ❌ 기타 파일들
- `builder-effective-config.yaml` - 빌드 설정 파일 (불필요)
- 기타 임시 파일들

## 업로드 방법

### GitHub 웹 인터페이스:

1. GitHub 저장소 → **Releases** → **"Create a new release"** 클릭
2. **Tag version**: `v1.0.1` (버전 앞에 `v` 추가)
3. **Release title**: `v1.0.1 - [변경사항 요약]`
4. **Description**: 변경사항 설명
5. **"Attach binaries"** 섹션에서 다음 2개 파일만 드래그 앤 드롭:
   - `이력서 적합도 평가 시스템 Setup 1.0.1.exe`
   - `latest.yml`
6. **"Publish release"** 클릭

### 파일 크기 제한

- GitHub Releases는 파일당 최대 2GB까지 업로드 가능
- 일반적으로 설치 파일은 100-200MB 정도이므로 문제없습니다

## 확인 사항

업로드 후 다음을 확인하세요:

1. ✅ 설치 파일(.exe)이 업로드되었는지
2. ✅ `latest.yml` 파일이 업로드되었는지 (가장 중요!)
3. ✅ Release 태그가 올바른지 (`v1.0.1` 형식)

## 자동 업데이트 작동 확인

사용자가 앱을 실행하면:
1. `latest.yml` 파일을 읽어서 현재 버전과 비교
2. 새 버전이 있으면 설치 파일(.exe)을 다운로드
3. 자동으로 설치 및 업데이트
